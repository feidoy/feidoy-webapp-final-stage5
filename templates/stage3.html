Classes:

Classes are like blueprints! The __init__() function creates space in our computer's working memory for instances of the class Turtle. In the example below, we are defining brad and britt, which are both instances of the class Turtle:

alex = turtle.Turtle
alexis = turtle.Turtle

So the class, Turtle, is like a basic blueprint within the module turtle that can be used to create multiple instances of the class.

An important thing to remember when using turtle is that the classes Turtle (turtle.Turtle) or Screen (turtle.Screen) must be capitalized


Creating CLasses:

Why create a class? Although many things are defined within the PSL, specifics aspects of a program that you want to create will not be covered and you have to create classes for yourself. Maybe you want to be able to call, see or in other ways use many different objects that are all of the same type. Like in the movie example, where we all know what a movie is and that many different movies exist.

So what should you think about when creating a class? What aspects of the class are things that all instances of the class have in common, and which of these would it be useful to be aware of, so that we can use them when creating instances of the class? Again, if we take the movie example, all movies(as far as I know) have a title, a storyline, a trailer, a poster etc. All of these are things that we may want to use for every instance of the class movie, but we don't want to waste time, space and common sense by re-creating all the aspects of a movie every time we define a new movie.

Creating the class

When creating the class, we use the Python keyword class, like this:

class Name_of_class():

This is followed by def __init__(), also known as the constructor, although not everyone agrees upon this(see above).

__init__() uses some terms, i.e. __init__(self, a, b, c, etc). More specifically:

self: this is for the object being created, so it points to the instance being created. In the movie example, this instance could be Star Wars, Toy Story or any other movie that we choose to create an instance for.

a, b, c, etc: are instance variables needed for the specific class we are creating. In a Food Truck example we have a Food Truck Name, a Food Truck Menu, and a Food Truck License Plate. These should have corresponding names, such as truck_name, truck_menu, truck_license etc, rather than having ambiguous names like a, b and c.

So if we want to create the class FoodTruck, and want to let it have the three instance variables above, we do it like this:

class Movie(): def __init__(self, truck_name, truck_menu, truck_license): self.truckname = truck_name self.menu = truck_menu self.license = truck_license

Module

A module is a file containing Python definitions and statements, essentially a Python program. The file is saved as a Python file, ending in .py. Modules and the functions defined in a module can be imported into other modules and be used there.

    Objects

    An object is a structure containing data (attributes) and coding functions (methods) that can use and manipulate the data. Objects can call, use and manipulate each other.

    Class

    Classes are a kind of blueprint that contain data and functions that control behaviour. They can be used to create multiple instances with different names but the same basic properties. In the screenshot, Movie is a class while Toy_Story and Avatar are instances of the class. Class vs Instance: A 'Class' is like a blueprint for a building while an 'instance' is like an ACTUAL building that was made from the blueprint.

    Method

    A method is a procedure, that does something inside a class or instance.
    An example is the procedure that opens a browser window, as seen in the screenshot above. see Instance Method below.
    The def __init__ that constructs/initiates the class is also a method.

    Constructor

    A constructor is a method, used to initiate and create memory for an object. The object being a new instance of a class.

    See more on the __init__ in Creating Classes further up in the notes.

    self

    Self is the fixed parameter in a class constructor that points to the class object being created.

    So if I have a class Movie, as seen above, self points to an instance of that class, wheter its Toy_Story or Avatar.

    Instances

    An instance is an object created using a class. The instance uses the properties and methods of the class, but can also have unique properties of it's own.

    Instance Variables

    Instance variables are variables defined within a class. These variables are used to define certain aspects of the of the class.

    The instance variables all point to self(mentioned above) and are probably, but not necessarily, unique for each instance.

    In the screenshot above, we can see that the instance variables tell us different things about Movie, like the title, storyline etc. And when we create Toy_Story or Avatar we define, and store in memory, the different values of the variables for each of these two instances.

    Instance Methods

    An instance method is a method within a class, used to enable each instance of the class to do the same thing. Much like instance variables, only with a specific function rather than just an integer or string value. Instance methods are special functions that belong to an 'instance' of a 'class.' They usually get or set information about the instance they are attached to.

    Again, the screenshot above gives us the webbrowser.open example, where this method can be used by both Toy_Story and Avatar, to open their trailer in a webbrowser.

    Class Variables

    A class variable is similar to an instance variable, in that it is shared by all instances of the class. BUT! It is defined outside any method's of the class. This means that it is an attribute of the class, but not an attribute of an instance of the class, compared to the instance variables above, where we can see that self.title and self.storyline are attributes of every instance of the class.

    In other words, a class variable is almost not a variable for the instances of the class, since it will always be the same for all instances. If it is changed and get's a new value of some kind, this value will change for all instances of the class. But it is, of course, a variable, since it's value can be changed.

    Python has some pre-defined class variables or class attributes, one of them being __doc__. This can be used to supply some kind of documentation for the class you have created, so that potential users can see what it does, quickly and easily. Adding a __doc__ class variable to your class can be done like this:

    class Nothing_special(object):

    """This class does nothing special"""

    Other pre-defined class variables are __name__, __dict__, __bases__ and __module__.
