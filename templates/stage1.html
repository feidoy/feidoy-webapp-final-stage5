HTML = Hyper Text Markup Language

Html is the language of the web. On The user end (with a browser) a request is sent to the internet, and communicates to a server through HTTP. The server sends back the HTML where the browser will interpret the HTML and display it on the web page.


Tags

HTML is made up of tags, and elements are complete from start of a tag, its string or contents, and the end of the tag. Tags that are "inline" keeps the text continuing on, including br.
Tags that are "blocks", place and invisible box around the contents that are inside them, like div and p tags.

CSS = Cascading Style Sheets.

CSS allows you to use syntax + rules to manipulate how elements look on a page. CSS is used to help style the page. It's manipulation can include font sizes, image border sizes, colors, shapes, position,etc.


DOM

Document Object Model - This is a Standard convention for interacting with elements in HTML. These are elements in a tree type listing.


Creating Structure

To create structure you can follow the tree method of by using DOM. Browsers turn tags to elements in a tree form with DOM. You can see the structures of these webpages through the developers tools (in Chrome). There you can see the tree structure with arrows that expand each element.

Elements- Elements run from the start of a tag, to the end of the tag. It can contain strings, content, numbers, images,and etc in between the tags.

Another thing to note about tabs is that they can have attributes, and values. For example, (div class="title") , where "class"is the attribute and "title" is the value.


Developer Tools

You can use the developer tool in chrome to look behind the scenes into the html of a webpage. You can see that the page has a tree structure, which allows you to dig deep into the elements hiding behind the arrows. Selecting an element can also reveal its CSS on the right side. The info can be used to scan the style of the page.

Alex's Section 2: Programming


Programming Notes:

A computer is a machine that can execute a program. With the right program, a computer can do any mechanical computation you can imagine. A program describes a very precise sequence of steps. Since the computer is just a machine, the program must give the steps in a way that can be executed mechanically. That is, the program can be followed without any thought. A programming language is a language designed for producing computer programs. A good programming language makes it easy for humans to read and write programs that can be executed by a computer.

Python is a programming language. The programs that we write in the Python language will be the input to the Python interpreter, which is a program that runs on the computer. The Python interpreter reads our programs and executes them by following the rules of the Python language.


Grammar:

Compared to a natural language, programming languages adhere to a strict grammatical structure. In English, even if a phrase is written or spoken incorrectly, it can still be understood with the help of context or other cues. On the other hand, in a programming language like Python, the code must match the language grammar exactly. The Python interpreter has no idea what to do with input that is not in the Python language, so it produces an error.

When programming language grammar is not followed the interpreter will return a SyntaxError message. This means that the structure of the code is inconsistent with the rules of the programming language.


Variables:

Variables make programs easy to understand. They can also use the same expression changing values of variables to compute different things. To assign variables, you'll need to create a 'name' and assign its 'value or expression' after placing an equal sign. (e.g. name = expression). Basically in a program you want to assign a name and its value. When you "print" a variables name, it will generate its value.

Variables can vary. = signs means assignment.ar:


Comments (#):

For comments, use hashtag #. The python interpreter will ignore this line. Commenting is used to help leave notes on code, which will help you in the future to remember how things were laid out, or even help others understand your code.


Expressions:

Expressions are one of the most basic instructions for a computer. The expression is made up of values and operators that evaluate down to single values. A single value can also be called an expression.


Data Types:

A data type is a category for values. There are 3 different data types. 1. Integers (1,3,-100)
2. Floating-point numbers (3.5, -6.73, 20.8889), and
3. Strings ("words or characters inbetween commas", "string", 'busy', 'example').


Assignment Statements:

Assignment Statments are variables, or inputs that are stored. It consists a variable name, and equal sign = (assignment operator) and value (which is to be stored as the variable name.
